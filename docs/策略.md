# Polymarket BTC 15 分钟二元预测市场交易策略

## 一、核心思想

**赚钱靠对冲价差，不靠方向预测。**

在便宜的一侧梯度接低建仓，靠对面配对买单把双边均价之和（sum_avg）压到 1 美元以下，使两侧的结算盈亏同时转正，锁定绝对利润。头寸扛到 15 分钟交割结算自然兑现。做市阶段全程只买不卖，唯一盈利出口是「两侧结算盈亏都达标 → 收手扛结算」。当单边裸露被行情拖入亏损时，启动对冲模块改善持仓结构。

二元期权结算刚性：胜方每股价值 1 美元，败方 0。

---

## 二、资金池

总资产 $V$ 在开盘时划分为四个独立资金池：

| 资金池 | 比例 | 作用 |
| --- | --- | --- |
| **备用金池** | 25% | 绝对禁止日常动用，仅极端防御下提供最后流动性。 |
| **核心做市池** | 15% | 做市阶段双边梯度铺单的唯一资金来源。 |
| **动态对冲池** | 22.5% | 动态对冲阶段（Maker 织网防守）的资金来源。 |
| **EV 对冲池** | 37.5% | EV 对冲阶段（Taker 扫盘）的资金来源。 |

资金重心在对冲：做市只用 15% 开小双边底仓，60% 留给对冲。

**全局现金哨兵**：任何模块下单前经资产审计器校验，可用现金不足时拒绝新开仓。

## 三、关键口径（全策略统一）

| 名称 | 公式 | 说明 |
| --- | --- | --- |
| **某侧结算 pnl** | $Q_{side} - C_{total}$ | 该侧若赢能拿回多少。$Q$=该侧净入仓股数（胜方每股值 1），$C_{total}$=双边累计净投入现金（含已发生手续费）。 |
| **某侧浮亏 pnl** | $Q_{side} \times bid_{side} - cost_{side}$ | 逐侧算，只对有持仓的侧算。用**本侧自己的成本**、用 **best_bid**（现在立即割肉能拿回多少）。 |
| **未配对保护成本** | $\max(0,\ Q_{up} - Q_{down}) \times \overline{P}_{up}$ | 主战场侧比对面多出来、没配对保护的裸露头寸现价值（用主战场侧成交均价）。 |
| **sum_avg** | $\overline{P}_{up} + \overline{P}_{down}$ | 双边成交均价之和。压到 1 以下则两侧结算 pnl 同时为正。 |

**为什么浮亏用 bid 不用 mid**：浮亏对冲是极端情况的底线防守，必须按「现在立刻强制市价割肉能拿回多少」算最坏情况，用盘口能立即吃单的 best_bid，绝不能用中间价掩盖流动性匮乏时的真实亏损。

## 四、阈值参数

| 参数 | 值 | 说明 |
| --- | --- | --- |
| **主战场阈值** | best_ask < 0.5 | 只在便宜侧建仓 |
| **利润 margin** | 0.02 | 配对价留出的利润空间 |
| **续挂步长** | 0.01 | 成交价下方续挂一档的偏移 |
| **利润锁定线** | +0.5%V | 两侧结算 pnl 都达标则收手 |
| **亏损触发线** | −2%V | 触发动态对冲 |
| **单边持仓门槛** | 3%V | 结算 pnl 触发的对侧成本门槛 |
| **最大敞口上限** | 11.25%V（动态对冲池的 50%） | UP 裸露敞口超此值则永久停铺 UP |

**+0.5% / −2% 不对称是有意为之**：做市利润来自微小价差积累，0.5% 是合理离场点；触发对冲意味着进入防御态，Taker 4% 费率 + 滑点使对冲成本极高，防守线太紧（如 −0.5%）会在正常震荡中被反复洗盘、被手续费抽干，必须留 −2% 容忍市场噪音。

**最大敞口为何挂钩对冲池的 50%**：建立「风险敞口」与「救火弹药」的硬链接。最极端单边行情下，系统永远保留 2 倍于当前裸露敞口的对冲资金可用，这是安全垫机制。

## 五、有限状态机

事件驱动。每个 tick 先过一遍**全局条件路由**（按优先级链从上到下裁决，见第十二节），再进入当前阶段的细节逻辑。

```
全局优先级链（每 tick 先判）：
  时间红线(<1min) > 熔断求生(Spread>30%) > EV对冲 > 动态对冲 > 核心做市

[ 场开始 ] ➔ 建仓态：铺初始三档梯度单
                 │
                 ▼ （首笔成交，哪怕 1 股）
            配对态（常驻）
                 │
                 ├─➔ [两侧结算 pnl 都 ≥ +0.5%V] ──────➔ 收手扛结算
                 ├─➔ [结算/浮亏 pnl 穿 −2%V] ─────────➔ 动态对冲
                 └─➔ [核心做市池耗尽] ────────────────➔ 挂机等待
                          │
                          ▼  动态对冲（Maker 织网，补亏损侧摊薄）
                          ├─➔ [两侧结算 pnl 都 ≥ +0.25%V] ──➔ 微利逃生·扛结算
                          ├─➔ [pnl ∈ (−2%V, +0.25%V)] ──────➔ 观察态（静默等行情）
                          └─➔ [敞口撞红线 / 双边负2次 / 末5min破线] ─➔ EV 对冲
                                   │
                                   ▼  EV 对冲（Taker 扫盘，顺势追优势方·方向翻转）
                                   ├─➔ [EV>0 / 概率跌破0.55 / 资金耗尽] ──➔ 收手扛结算
                                   └─➔ [概率不在甜区] ───────────────────➔ 装死(Skip)

任意阶段：
  Spread>30% ──➔ 熔断求生态（CancelAll·稳定5s后重走全局路由）
  剩余<1min ──➔ 时间红线（无条件 CancelAll·锁死 Settlement_Wait）
```

## 六、建仓态

1. 选 best_ask < 0.5 的一侧为主战场，两侧皆满足取更低者（例：开局 UP 建仓）。**一轮生命周期内绝对不换边**——状态机需要锚点，频繁切换会变成双边裸奔。
2. 以该侧 best_ask 为基准，向下平铺三层非饱和阵型：
   - 1 层：best_ask − 0.01，动用核心做市池 2%
   - 2 层：best_ask − 0.02，动用核心做市池 3%
   - 3 层：best_ask − 0.03，动用核心做市池 5%
3. 价格向下取整（不抬买价），股数向下量化后校验最小量（5 股且 ≥ $1），不够的档跳过（不上调，守资金纪律）。
4. 首笔成交（哪怕 1 股）立刻单向跃迁至配对态。

初始三档满打满算占 1.5%V（核心做市池 15% × 10%）。

## 七、配对态

常驻状态，直到利润锁定、对冲触发或最后 5 分钟。

### 主战场侧（UP）成交 → 触发重算

1. **检查最大敞口**：
   - 未超 11.25%V → 在更低价（成交价 − 0.01）补一档 UP 限价单，占核心做市池 2%。
   - 已超 11.25%V → 撤销所有 UP 未成交挂单，**本场永久停铺 UP**（哪怕之后 DN 配对成交把敞口降回去也不恢复，防趋势盘反复抖动）。
   - 敞口判定：未配对保护成本 + UP 活跃挂单总金额 + 拟发新单金额 ≤ 11.25%V。
2. **DN 侧铺配对单**：
   - 价格 = $\min(1 - \overline{P}_{up} - 0.02,\ ask_{down} - 0.01)$
   - 数量 = 主战场总持仓 − 对面已有总持仓（目标摽齐，差额 ≤ 0 不下单，根除滚雪球）

取 min 让配对价永远 ≤ 对面 ask − 0.01，既能立即排队又不牺牲利润，消除了「配对价高于 ask 只能挂起等待」的情况。

### 配对侧（DN）成交 → 不触发重算

配对侧成交意味着风险闭环和利润兑现。**绝不触发重新铺设主战场逻辑**（防无限套娃建仓），仅做两件事：
1. 更新本地账本和均价。
2. 触发检查全局「利润锁定」或「对冲触发」阈值。

### 精细订单管理（UP 成交后 DN 配对单的处理）

UP 成交导致 UP 均价变化，需重算 DN 最新配对价，对比当前在盘口排队的旧配对单：

- **配对价变化**（`|最新配对价 − 旧配对价| ≤ 0.01`）：不撤单。微小价差不足以覆盖重新排队的成交率损失，让旧单继续排队保住 time priority。
- **配对价变化 > 0.01**：撤旧单、重挂新单。均价已被大幅摊低，旧单挂得太高（利润空间过大难成交）或太低（成交会导致利润不达标），必须修正。
- **缺口变大但配对价 ≤ 0.01**（如原差 100 股、现差 150 股）：不撤原 100 股单，在相同价格追加发送 50 股新单。原排队优势完美保留，只在队尾多加 50 股。

## 八、退出条件

| 触发 | 动作 |
| --- | --- |
| **利润锁定** | 两侧结算 pnl 都 ≥ +0.5%V → 撤所有单收手扛结算（唯一盈利出口） |
| **结算 pnl 穿线** | 任一侧结算 pnl ≤ −2%V **且** 引发亏损的对侧持仓成本 > 3%V → 动态对冲 |
| **浮亏 pnl 穿线** | 任一侧浮亏 pnl ≤ −2%V → 动态对冲 |
| **核心做市池耗尽** | 拟挂金额 + 已挂金额 + 已成交金额 ≥ 15%V，或剩余不够最小单 → 挂机等待（不进对冲） |
| **最后 5 分钟** | 亏损大侧「结算 pnl × 该侧概率 ≤ −2%V」→ 强制 EV 对冲（Taker 扫盘）；否则收手扛结算 |

**最后 5 分钟动态对冲彻底失效**：决战垃圾时间波动剧烈、流动性极差，Maker 单大概率排不到队或瞬间变深海死单。所以只走 EV 对冲（IOC Taker 暴力扫盘）或收手躺平。

## 九、动态对冲模块

被动急救态。本质是**戴双闸的有限赌反弹**：在亏损大侧低位织 Maker 网摊薄均价、把 sum_avg 拉回 1 以下。两道死闸（敞口红线 + 双边负 2 次）兜住单边趋势下的马丁格尔失控——撞任一闸都强制升级 EV，绝不无限接飞刀。

### 进入

配对态满足以下任一条件触发（与第八节退出条件一致）：
- 任一侧结算 pnl ≤ −2%V **且** 引发亏损的对侧持仓成本 > 3%V
- 任一侧浮亏 pnl ≤ −2%V

进入时：
- 撤销做市阶段所有挂单。
- 做市态「敞口超 11.25%V 永久停铺 UP」的 flag 在对冲态**不生效**（那是主动出击的纪律，被动急救不受此限）。
- 开启「双边都负」独立计数器。

### 补哪边

补 **PnL 较差（亏损大）侧**。亏损大侧通常是高位站岗（持仓均价远高于市价），在该侧当前低位 best_ask 织 Maker 网低位接盘，强行摊薄该侧均价，把 sum_avg 拉回 1 以下。

### 单步织网

没有「轮」的概念，一步一步持续进行。单步资金 = 对冲池**当前剩余**的 7.5%，铺三层 Maker 梯度单：
- 1 层：best_ask − 0.01，占本步资金 40%
- 2 层：best_ask − 0.02，占本步资金 30%
- 3 层：best_ask − 0.03，占本步资金 30%

**不做摽齐 cap**：第一阶段配对态要封顶是因为那是主动套利、买多了造成新敞口；动态对冲是被动自救、目的是强行摊薄，按预定资金计划坚决执行，只受最大单边敞口红线约束。

### 单步生命周期（三种结束）

- **全成交**：本步 Maker 全部成交（理想），记时间戳 s1。
- **超时（3 秒）**：到达最长生命周期，不管成没成交强行结束。由**本地 wall-clock 定时器**兜底触发——发单瞬间启动 3 秒异步定时任务，全成交则取消；3 秒到仍未全成交则强制介入：按撤单规则撤未成交单、宣告本步超时结束、记 s1。保证缺 tick 时不死锁。
- **价格偏离**：挂了 best_ask − 0.01 的 Maker，但市场暴跌，best_ask 瞬间偏离挂单价超 0.03，Maker 已成深海死单毫无防守意义，立即判本步失效结束。

### 冷却（1 秒）

两步之间 1 秒冷却，用**时间戳卡点被动校验**（不起定时器主动唤醒）。本步结束记 s1，此后的 tick 若 `Now − s1 < 1s` 直接 return；满 1s 后的某个 tick 才真正发起下一步。实际 >1s（要等下一个 tick 到来）。

### 每步开始前

检查该侧未成交 Maker 单偏离 best_ask 是否 > 0.03，是则精确撤销。深海死单会占用对冲预算池额度，不撤就算不出下一步的钱。

### 状态流转（以结算 pnl 为基准）

| pnl 区间 | 动作 |
| --- | --- |
| pnl ≤ −2%V | **继续对冲**（每步 1 秒冷却） |
| (−2%V, +0.25%V) | **观察态**：停发新对冲单、不撤已挂单、等行情自然修复。跌破 −2%V 重启对冲 |
| pnl ≥ +0.25%V | **微利逃生** → 撤所有单转结算态扛交割 |

观察态只是「暂停织网动作」，不是「免疫升级」：双边负计数、最后 5 分钟、敞口检查仍在跑，满足照样跳 EV 或扛结算。

### 升级 EV 的三条路（任一触发立即跃迁）

1. **敞口撞红线**：亏损侧补单前算出「该侧持仓成本 + 活跃挂单 + 拟发新单」将超 11.25%V → 拒绝下单 + 强制跳 EV（哪怕没满足双边负 2 次）。温和 Maker 防守已走到尽头，交给 EV 用最后弹药 Taker 扫盘止血。
2. **双边都负 2 次**：进入动态对冲即开计数器，任何时刻发现 UP 与 DN 结算 pnl 同时 < 0 则 +1，达 2 立即跳 EV。双边同时亏损说明 sum_avg > 1 且两侧都被高位套牢，Maker 织网已失效。
3. **最后 5 分钟**：亏损大侧「结算 pnl × 该侧概率 ≤ −2%V」→ 强制 EV；否则收手撤单扛结算。最后 5 分钟波动剧烈、流动性极差，Maker 单大概率排不到队或瞬间成深海死单，动态对冲彻底失效。

### 其他退出

- **微利逃生**：两侧结算 pnl 都 ≥ +0.25%V → 撤所有单收手扛结算。
- **资金耗尽**：对冲池剩余不够最小下单量或金额 → 收手（落到观察态/扛结算）。

### 动态对冲阶段的线

| 参数 | 值 |
| --- | --- |
| 微利逃生线 | +0.25%V |
| 观察线 | −0.5%V |
| 亏损触发线 | −2%V |

## 十、EV 对冲模块

终极止血态。动态对冲（Maker 逆势接飞刀）的两道死闸被冲破，说明逆势防守彻底破产、趋势已确立。此时系统**认输止血**：不再接亏损侧飞刀，而是从「马丁格尔逆势摊薄」彻底翻转为「动量顺势跟随」——动用最大的一笔钱（EV 对冲池 37.5%V），顺势 Taker 追买**真正会赢的优势方**（概率 p > 0.5 那侧），靠交割时拿到的刚性胜方收益（每股 1）覆盖劣势方的巨额沉没成本。**方向翻转是 EV 对冲的核心灵魂。**

### 数学期望公式

$$EV = p \times up\_settle\_pnl + (1-p) \times down\_settle\_pnl$$

- $p$ = UP 侧 Mark Price（盘口中间价，近似 UP 胜出概率）。
- up/down_settle_pnl 用**结算 pnl 口径**（$Q_{side} - C_{total}$，含双边总成本），绝不用浮亏口径——EV 算的是「现在花钱吃货，到 15 分钟交割时系统整体的期望收益」。

### 进入

从动态对冲三条路之一跃迁而来（敞口撞红线 / 双边负 2 次 / 最后 5 分钟破线）。进入时立刻 **Cancel All** 撤销所有挂单，盘面清零（之后是「空仓挂单、只有已成交持仓」状态）。

### 战略进入 ≠ 战术开火（关键解耦）

「进入 EV 状态」和「打出 EV 子弹」是两码事。进来后先评估优势方概率，只有落在甜区才真正出手（HedgeDecision::Fire），否则装死（HedgeDecision::Skip，不发任何 API 请求）。

### 出手甜区（按剩余时间分段）

| 剩余时间（TTE） | 优势方概率出手区间 |
| --- | --- |
| > 5min | [0.60, 0.75] |
| 5min ~ 1min | [0.75, 0.85] |
| < 1min | 强制收手（不出手） |

越临近交割流动性越差，要求的确定性越高（甜区上移）。

**概率高于甜区上限（如 >5min 时 >0.75）→ 装死（Skip），不做冤大头**。概率飙高说明价格极贵（滑点巨大）+ 市场共识极度趋同，用 4% Taker 追买盈亏比恶劣。等它掉回甜区再打；若一去不回头硬挺到结算，就直接扛到结算收割（认栽或听天由命）。

**概率落在 [0.55, 0.60) 迟滞区 → 装死等市场审判**。0.58 的胜率不足以覆盖 4% Taker 费，强行出手 EV 为负、亏更多。等概率涨过 0.60 开火，或跌破 0.55 触发反转退出。

「装死」只是引擎这一刻不发 API 请求；FSM 没死——每个 BookUpdate 都重算 EV、概率区间、反转线，满足出手或退出条件就在下一个 tick 瞬间响应。

### 单步

- 没有「轮」的概念，一步一步持续进行。
- 单步金额 = **EV 对冲池当前剩余**的 25%。
- **强制 IOC 模式**：发价格上限为 0.85 的限价单，撮合引擎吃掉 best_ask 到 0.85 之间所有流动性，未成交部分立即撤销。绝不允许挂成 Maker 被动等成交。
- 单步生命周期：IOC 在同一原子事务里返回 Fill（成交）+ Cancel（未成交）回报，收到回报本步即自然闭环，不需要 wall-clock 定时器兜底。
- 两步之间 **2 秒冷却**（时间戳卡点被动校验）。

### 退出（任一满足即撤所有单扛结算）

- **EV > 0**：期望转正，目的达成。
- **利润重锁**：两侧结算 pnl 都 ≥ 利润锁定线（通常不会发生，因进入时已撤所有挂单）。
- **概率跌破 0.55**：判定期望值反转，立即收手。
- **资金耗尽**：EV 池剩余不够最小下单量或金额。
- **剩余 < 1min**：时间红线强制收手（见第十二节）。

## 十一、熔断求生态（Circuit_Breaker_Survival）

流动性崩溃时的全局保护。不属于任何交易阶段，是凌驾于做市/对冲之上的安全中断。

### 触发指标：买卖价差率

$$Spread\_Ratio = \frac{best\_ask - best\_bid}{best\_bid}$$

正常 15 分钟盘 Spread_Ratio 维持极低（约 0.01~0.03）。**任一侧** Spread_Ratio > 30% 即触发全局熔断（例：bid 掉到 0.20、ask 挂 0.55）——说明理性做市商已撤单跑路，流动性崩溃。

**为何任一侧都看**：UP/DN 价格强镜像耦合（UP + DN ≈ 1）。非主战场侧突然 spread 爆宽不是孤立事件，意味着整个定价机制崩溃（预言机故障 / 巨鲸撤资 / 内幕泄露）。流动性崩溃是全局信号，任一侧异常都必须立刻全局熔断，否则会被单边毒流穿透。

### 执行动作

立刻 Cancel All，FSM 切换至熔断求生态。

### 防抖恢复（迟滞）

必须 Spread_Ratio < 10% **且持续稳定 5 秒以上**才解除熔断。不加这 5 秒迟滞，机器会在 spread 于 30% 上下横跳时疯狂发单又撤单，浪费 API 速率额度甚至触发风控封禁。

### 恢复即重新评估（带记忆的全局重评）

熔断恢复**不是回到断点继续**（闪崩期间价格、概率、TTE 已剧变，刻舟求剑会出错），而是：
1. **保留状态变量**：双边亏损计数、总持仓成本、各资金池剩余预算等底层计数器和标记全部保留。
2. **挂单已清零**：熔断时已 Cancel All，恢复时是「空仓挂单、只有已成交持仓」。
3. **重走全局条件路由**（见第十二节）：把最新盘面切片丢进路由从上到下重判，决定进结算 / EV / 降级动态对冲 / 退回做市。

## 十二、全局规则

### 绝对优先级链（Absolute Priority Chain）

任何 tick 都先过一遍优先级链，从上到下裁决，高优先级无条件压制低优先级：

```
时间红线 (<1min) > 熔断求生 (Spread>30%) > EV 对冲 > 动态对冲 > 核心做市
```

这是**全局条件路由（Global Evaluation Router）**的实现：所有「同时满足多个条件」的歧义都靠这条链裁决。

### 时间红线（<1min 硬中断）

剩余时间 < 1min 时拥有系统最高优先级。无论当前处于做市、动态对冲、EV 对冲还是熔断求生的哪个阶段，**无条件切断当前执行流，立刻 Cancel All，状态锁死在 Settlement_Wait（等待结算态）**，再不恢复。

生存哲学：此时任何挣扎的期望值都是负的，带现有持仓硬扛到结算（哪怕亏损）也绝对优于在最后几十秒被超大滑点双杀。

时间红线 > 熔断恢复：若熔断 5 秒防抖期未数完就到了 <1min，时间红线强行接管、丢弃 5 秒计时器、直接锁进 Settlement_Wait。

### Maker 挂单价规则（做市态 + 动态对冲态）

取消旧的 pending_orders 机制。**Maker 单挂单价**规则：
- 挂单价 < best_ask → 直接挂。
- 否则 → 以 best_ask − 0.01 挂。

目的：防止做市时意外成为 Taker 被收 4% 手续费。**仅适用做市态和动态对冲态的 Maker 单**，EV 阶段的 Taker 单不受此约束（EV 本就主动做 Taker，直接 IOC 设保护上限价 0.85）。

---

## 十三、关键红线

- **绝不主动止损**：除熔断求生与时间红线外，做市阶段持仓严禁割肉，靠对冲调整结构或扛到结算。
- **绝不触碰备用金池**：EV 对冲也只动用 EV 对冲池，25% 备用金池绝不动用。
- **高频账本高可用**：每笔成交原子级落盘（WAL），panic 重启可从本地 JSONL 完美复原。均价为精确值不做舍入。

---

## 十二、手续费模型

- Taker 费率 ~4%（按 4% 计提，略偏保守），Maker 费率 0%。
- 扣费口径：到手股数扣减（下单 100 股价 0.5 费率 4% → 实付现金 50、净入仓 96 股）。
- 费率可配置，由 `FeeModel` 统一管理。
