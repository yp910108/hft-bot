# Polymarket 短周期二元预测市场自动化交易策略

## 一、核心思想与资金防御矩阵

### 1. 核心交易哲学

**赚钱靠对冲价差，不靠方向预测。**
系统通过在两边进行动态梯度接低建仓，利用对立面配对买入的订单，将双边持仓的加权均价之和压缩至 1 美元以下，使两边的条件胜出盈亏同时转为正数，锁定绝对利润。所有头寸一律扛到交割结算期自然兑现。当单边极端行情导致系统暴露严重方向性风险时，启动自适应对冲模块改善持仓结构。

### 2. 资金池精细化治理

总资产（$V$）在开盘时被严格划分为三个独立运作的子资金池：

| 子资金池 | 划拨比例 | 作用 |
| --- | --- | --- |
| **备用金池 (`Reserve_Pool`)** | **25%** | 绝对禁止日常动用，仅在极端对冲防御状态下提供最后的流动性注入。 |
| **核心做市池 (`Grid_Maker_Pool`)** | **52.5%** | 维持日常双向非对称网格的 Maker 梯度铺单。 |
| **动量对冲池 (`Hedge_Attack_Pool`)** | **22.5%** | 对冲阶段的 Taker + Maker 梯度追买资金来源。 |

**全局可用现金安全哨兵 (Cash Guard)**：任何执行模块在提交订单前，必须通过资产审计器校验。若可用现金 $< \text{cash\_guard\_ratio} \times V$（默认 = reserve 比例），拒绝一切新开仓。cash_guard_ratio 必须 >= reserve 比例。

## 二、条件盈亏计算与对冲触发边界

### 1. 条件盈亏口径 (Conditional PnL)

$$UP\_Win\_PnL = Q_{up} - C_{total}$$
$$DOWN\_Win\_PnL = Q_{down} - C_{total}$$

- $Q_{up}$/$Q_{down}$：当前持有的 UP/DOWN 股数（扣费后的净入仓股数）。
- $C_{total}$：双边累计投入的净总成本（含已发生手续费）。
- **利润锁定判定**：$\min(Q_{up}, Q_{down}) > C_{total}$ 且两边 PnL 均 ≥ profit_target。

### 2. 对冲触发边界

双因子复合判定，防止开局首次成交导致的库存短暂失衡引发虚假报警：

$$\text{Trigger} = (\text{Side\_PnL} \le -\text{hedge\_loss\_trigger}) \ \mathbf{AND}\ (\text{total\_qty} \ge \text{hedge\_min\_qty})$$

- 两边同时穿线且 `both_sides_negative` → 直接跳 EvHedging（补哪边都没用）。
- 仅单边穿线 → 进入 DynamicHedging。

## 三、有限状态机（FSM）流转架构

系统采用订单成交事件驱动架构，所有状态流转由交易所异步回报的成交事件触发。

```
[ 场开始 ] ➔ Initialization 部署初始梯度单
                 │
                 ▼
     RangeBoundMaking (梯度接低循环) ───➔ [利润锁定] ───➔ FinalSettlement
                 │
                 ├─➔ [单边穿线]
                 │         ▼
                 │   DynamicHedging (梯度追买瘸腿侧)
                 │         │
                 │         ├─➔ [利润锁定] ───➔ FinalSettlement
                 │         ├─➔ [两边均负 × 2] ───➔ EvHedging
                 │         └─➔ (继续追买直到利润锁定或步数/预算耗尽)
                 │
                 └─➔ [双边同时穿线且两边均负] ───➔ EvHedging
                                                      │
                                                      ├─➔ [EV ≥ 0] ───➔ FinalSettlement
                                                      └─➔ (继续追买优势方)
```

## 四、梯度接低模块 (Gradient Low-Catching)

### 1. 动态盘口扫描与非饱和布阵

1. 选择 $best\_ask < 0.5$ 的一侧作为本轮做市主战场（两侧皆满足取更低者）。
2. 以该侧 `best_ask` 为基准，向下平铺三层非饱和防御阵型：
   - 1层：$best\_ask - 0.01$，动用核心做市池的 2%
   - 2层：$best\_ask - 0.02$，动用 3%
   - 3层：$best\_ask - 0.03$，动用 5%

### 2. 跨侧配对与追低逻辑

每当本边 Maker 买单成交：
1. 更新本边持仓与均价
2. 在成交价下方续挂追低（follow_offset=0.01，占比 2%）
3. 撤销对面全部活跃挂单
4. 计算对面配对价 = $1 - \text{本边均价} - \text{min\_profit\_margin}(0.02)$
5. 配对量 = 对面目标持仓 - 对面当前持仓（摽齐差额，只补不超）
6. 若配对价 < 对面 Ask → 直接挂 Maker；否则 → Pending 等 Ask 跌下来

### 3. 防滚雪球机制

- 仅主战场侧的主动接低成交触发跨侧重算
- 对面配对成交不触发（断开「配对又触发配对」的正反馈）
- 世代号隔离：旧世代成交入账但不触发重算

## 五、动态对冲模块 (Dynamic Hedging)

**目标：持续追买瘸腿侧，直到双边 PnL 都为正（利润锁定）。**

对冲不是"补到不穿线就停"，而是"补到赚钱才停"。不穿线只是触发对冲的入口条件，利润锁定才是终态。

### 当前实现（MVP — 待改进）

1. **确定瘸腿侧**：PnL 穿透亏损线的一侧；双侧同时穿透取更亏侧。
2. **单笔 Taker 追买**：单步预算 = 动量对冲池 × step_fraction(0.2)，以该侧 best_ask 即时吃单。
3. **摽齐封顶**：单步量 = min(预算可买量, 对面持仓 - 本侧持仓)。
4. **限流**：每个行情 tick 最多打 1 步 Taker（防止同价连环打光）。
5. **预算硬上限**：对冲池剩余额度跨阶段不重置，花完就停。
6. **停手条件**：步数达上限(5)、预算花完、或利润锁定。

### 待改进方向（下一步）

当前问题：单笔 Taker 一次吃一大笔，价格集中在一个 best_ask，没有分散。且对冲阶段没有 Maker 挂单参与。

改进思路：
1. **对冲也分层梯度**：像初始布阵一样，在瘸腿侧向下铺多档买单（价格分散、拉低均价）。
2. **Maker + Taker 混合**：最近的一档用 Taker 确保成交（应对趋势），更低档用 Maker 等更好价格（省手续费）。
3. **停手条件改为 PnL 驱动**：不再看"穿不穿线"，而是看"双边是否都为正"。瘸腿侧 = PnL 更小的那侧（不管穿没穿线），持续追买直到利润锁定。
4. **对冲完一轮后继续做市**：如果梯度挂单被吃完了、对冲把 PnL 拉回来但还没锁定利润，应按当前价格重新铺梯度单继续接低。

### "两边均负"的判定时机

"两边 PnL 同时为负"只看**已成交的持仓**（账本里的 qty 和 cost），跟簿上的 Maker 挂单无关。
判定时机：每笔成交入账后重新评估 FSM。不管这笔成交来自 Maker 撮合还是 Taker 即时成交，入账后看一眼——两边都负就 count+1，连续两次升级到 EvHedging。

## 六、EV 对冲模块 (Expected Value Hedging)

动态对冲中连续两次"双边 PnL 同时为负"后转入本阶段。

$$EV = p \times UP\_Win\_PnL + (1-p) \times DOWN\_Win\_PnL$$

其中 $p$ = Up 侧 Mark Price（盘口中间价，近似 Up 胜出概率）。

### 执行逻辑

1. 若 $EV \ge 0$：撤单停手，进入 FinalSettlement。
2. 若 $EV < 0$：Taker 追买优势方（$p > 0.5$ 则追 Up，否则追 Down），纯预算封顶（无摽齐缺口限制）。
3. 步数上限独立计数(5)，达上限停手。
4. 累计投入硬上限：对冲池剩余额度（跨阶段不重置）。

### 阶段切换衔接规则

**Dynamic → Ev 切换时：CancelAll 清场。** 原因：
- Dynamic 阶段铺的是瘸腿侧的梯度单，Ev 阶段追买的可能是另一侧（按 mark_price 判断优势方）
- 即使方向相同，Ev 的预算分配和量计算逻辑不同
- 留旧单会导致成交归属混乱

同理，任何阶段切换（包括对冲结束回到做市）都应先 CancelAll 再从干净状态开始。

## 七、收手与终局

| 触发条件 | 执行动作 |
| --- | --- |
| **利润锁定** | 双边 PnL 均 ≥ profit_target → CancelAll + 停手扛到交割 |
| **EV 归正** | $EV \ge 0$（EV 对冲阶段）→ 撤单停手 |
| **对冲预算耗尽** | 对冲池剩余 < 单步预算 → 停止 Taker，扛到交割 |
| **对冲步数耗尽** | Max_Taker_Steps 达上限 → 停止 Taker，扛到交割 |
| **猴市熔断** | 极端多空双杀 → 一键清仓关机 |

## 八、关键红线约束

- **绝不主动止损**：除 ChopMarketShutdown 外，持仓严禁割肉，靠对冲调整结构或扛到结算。
- **高频账本高可用**：每笔成交原子级落盘（WAL），panic 重启可从本地 JSONL 完美复原。均价为精确值不做舍入。

---

## 九、手续费模型

- Taker 费率 ~4%（略偏保守计提），Maker 费率 0%。
- 扣费口径：到手股数扣减（下单 100 股价 0.5 费率 4% → 实付现金 50、净入仓 96 股）。
- 费率可配置，由 `FeeModel` 统一管理。
