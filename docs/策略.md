# Polymarket BTC 15 分钟二元预测市场交易策略

> 本策略高保真复刻链上地址 0xe00740bc 的做市库存管理打法（402 场，场均 +11.42，胜率 53%）。
> 完整分析见 `docs/0xe00740bc 策略分析.md`。

---

## 一、核心思想

**赚钱靠高频做市的循环价差，不靠方向预测。**

在两侧同时挂 Maker 单持续做市：便宜就买、涨了就卖，每笔买入独立跟踪、涨到止盈幅度就挂 Maker 卖单收割微利。反复循环回收现金，把净持仓的等效成本（sum_avg）压到 1 美元以下。最终保留双边净持仓扛结算，无论哪方赢都大概率盈利。

二元期权结算刚性：胜方每股价值 1 美元，败方 0。若两侧净持仓的等效均价之和 < 1，则无论谁赢，赢家侧的结算回报都能覆盖总净投入 → 稳定盈利。

**盈利有两个来源，叠加生效：**
1. **场内循环差价**：每笔止盈 +0.05~0.12，回收约 57% 的买入资金，同时压低净持仓成本。
2. **净持仓 sum_avg < 1**：收手时保留的双边净持仓，结算时兑现最后的利润。

**这套策略的燃料是波动**：价格必须运动才有止盈机会。低波动场（波动 < 0.4）是天然短板，需要过滤或减仓。

## 二、关键口径（全策略统一）

| 术语 | 定义 |
| --- | --- |
| **UP / DN** | 二元市场的两个方向 token（涨 / 跌），结算时赢家 =$1、输家 =$0 |
| **价格** | 某侧 token 的市场价，UP_price + DN_price ≈ 1（两侧价格互补） |
| **bid / ask** | 盘口最优买价 / 卖价，spread 稳定为 1 tick（0.01） |
| **tick** | 最小价格单位 = 0.01 |
| **一笔（lot）** | 单次下单，固定股数（默认 10 股），金额随价格浮动 |
| **净持仓（net_qty）** | 某侧累计买入股数 − 累计卖出股数 |
| **净成本（net_cost）** | 某侧累计买入花费 − 累计卖出回收 |
| **净均价（net_avg）** | 某侧 net_cost / net_qty，即净持仓的等效买入价 |
| **sum_avg** | UP 净均价 + DN 净均价，< 1 则结算稳赚 |
| **场内进度（progress）** | 已过时间 / 900s，∈ [0, 1]，用于分段调参 |
| **止盈幅度（tp）** | 卖出价 − 该笔买入价，达到即挂 Maker 卖单 |
| **止损幅度（sl）** | 该笔买入价 − 当前 bid，超过即割肉 |

## 三、策略生命周期（三阶段）

策略在每场 15min 周期内经历三个阶段，靠场内进度和持仓状态自然过渡，无硬性切换线。

```
┌─ 阶段 1：开场建仓（progress 0 ~ 8%，约前 70s）──────────────────┐
│ 双边同时挂 Maker 买单铺底仓                                       │
│ 优先便宜侧，单笔 10 股，两侧交替铺单                              │
│ 目标：前 70s 双边各建约 3~5 笔底仓                                │
└───────────────────────────────────────────────────────────────────┘
       ↓
┌─ 阶段 2：循环做市（progress 8% ~ 83%，约 70s ~ 750s）─────────────┐
│ 主体阶段，持续做市循环：                                          │
│   ① 持续在两侧挂 Maker 买单（便宜侧优先，受库存约束限制）        │
│   ② 每笔买入成交后，在 +tp 价位挂 Maker 卖单（逐笔止盈）        │
│   ③ 卖单成交 = 锁定利润 + 回收现金；未成交继续等                  │
│   ④ 某笔浮亏超过 sl → 撤卖单、Taker 割肉止损                     │
│   ⑤ 库存约束：单侧净持仓超上限则暂停该侧买入（防接飞刀）        │
└───────────────────────────────────────────────────────────────────┘
       ↓
┌─ 阶段 3：收手变现（progress 83% ~ 100%，约 750s ~ 900s）───────────┐
│ 停止新开买单，专注变现和对称                                      │
│   ① 继续挂止盈卖单收割浮盈                                        │
│   ② 若两侧净持仓不对称，补买少的一侧使其接近对称                  │
│   ③ 目标：sum_avg < 1 且两侧净持仓量接近相等                     │
│ 距结算约 30s 后完全停手，保留净持仓等结算                        │
└───────────────────────────────────────────────────────────────────┘
```

## 四、详细决策规则

### 阶段 1：开场建仓

**入场时机**：数据可用即入场（我们的逐秒数据从第 6 秒起，尽早入场）。

**建仓动作**（每个决策 tick 执行）：
- 在 UP 和 DN 两侧各挂一个 Maker 买单
- 挂单价：该侧 bid（排队等成交，纯 Maker）
- 单笔股数：10 股（固定）
- 优先级：若两侧价格不对称，优先买便宜侧（price 更低的一侧）
- 节奏：不追价，挂单等成交，成交后再挂下一笔

**结束条件**：progress ≥ 8%（约 70s）或双边各已建 ≥3 笔底仓，转入阶段 2。

**参数**：
| 参数 | 默认值 | 说明 |
| --- | --- | --- |
| 入场秒数 | 6s | 数据起始即入场 |
| 建仓期时长 | 70s | progress 8% |
| 单笔股数 | 10 | 固定 |
| 建仓目标笔数 | 3~5/侧 | 铺底仓 |

### 阶段 2：循环做市（核心）

**这是策略的主体，包含买入、止盈、止损三个并行的决策流。**

#### 2.1 买入决策（补仓）

在两侧持续挂 Maker 买单，条件：
- 挂单价 = 该侧当前 bid（Maker 排队）
- 该侧净持仓 < 库存上限（见 2.4）
- 优先便宜侧：两侧都可买时，买 price 更低的一侧
- 单笔 10 股

买入成交后 → 立刻为这笔买入登记一个止盈卖单（见 2.2）。

#### 2.2 逐笔止盈（LIFO 特征）

**每笔买入独立跟踪，成交后立即在高位挂 Maker 卖单等收割。**

- 止盈挂单价 = 该笔买入价 + tp（止盈幅度）
- 卖单类型：Maker（挂在高位等对手来吃，0 费率）
- 卖出股数 = 该笔买入的股数（逐笔对应，不按持仓比例）
- 卖单成交 → 该笔利润锁定，现金回收，该笔从跟踪列表移除

**止盈幅度 tp 随场内进度递增**（从数据提取）：

| 场内进度 | tp（止盈幅度） |
| --- | --- |
| Q1 (0~25%) | +0.05 |
| Q2 (25~50%) | +0.06 |
| Q3 (50~75%) | +0.08 |
| Q4 (75~100%) | +0.10 |

后期持仓时间更长、价格运动更大，故止盈目标放大。

#### 2.3 逐笔止损

**某笔买入浮亏超过 sl，撤掉其止盈卖单，Taker 割肉。**

- 触发：该笔的当前浮亏 =（该笔买入价 − 当前该侧 bid）≥ sl
- 动作：撤止盈卖单 → 以当前 bid Taker 卖出该笔股数
- 止损用 Taker（要立刻离场，不等），接受 spread 损失

**止损幅度 sl 随场内进度放宽**：

| 场内进度 | sl（止损幅度） |
| --- | --- |
| Q1 (0~25%) | 0.04 |
| Q2 (25~50%) | 0.04 |
| Q3 (50~75%) | 0.05 |
| Q4 (75~100%) | 0.07 |

#### 2.4 库存约束（第二版可选项）

> 第一版纯复刻不启用此约束。第二版根据回测结果评估是否开启。

0xe00740bc 不刻意对冲，但我们的 BTC 数据 79% 是单向趋势，纯做市可能在趋势中不断接下跌侧的飞刀。待第一版回测后，若趋势场亏损严重，启用库存上限约束：

- 单侧净持仓 ≤ 库存上限（默认 200 股）→ 超过则暂停该侧买入
- 双侧净持仓差 ≤ 不平衡上限（默认 100 股）→ 超过则只准买少的一侧
- 约束只限制买入，不强制对冲（对称仍靠循环自然达成）

### 阶段 3：收手变现

**progress ≥ 83%（约 750s）进入，停止常规补仓。**

- **停止新开买单**：不再主动补仓
- **继续挂止盈卖单**：已有持仓的止盈卖单继续挂着收割
- **对称补仓（第二版可选项）**：第一版不启用。第二版评估是否在收手期补买少的一侧修复不对称（修复"sum_avg<1 却因持仓不对称而亏"的第三类亏损，占亏损场 29%）
- **完全停手**：距结算 ≤30s（progress ≥ 96.7%）后不再下任何单，保留净持仓等结算

## 六、回测建模约定

### Maker 成交判定（第一版：穿越即成交）

- **Maker 买单**：挂单价 ≥ 对手 ask 时，判定成交（价格穿越）
- **Maker 卖单**：挂单价 ≤ 对手 bid 时，判定成交（价格穿越）
- **Taker（止损）**：以当前对手价立即成交

第一版用穿越即成交跑通逻辑、验证策略正确性。此模型偏乐观（高估成交率），成交率的队列建模作为后续独立优化项精化。

### 费率

- Maker：0%
- Taker：约 3%，按 4% 计提（已核实，扣股数方式：下单 100 股价 0.5 费率 4% → 实付现金 50、净入仓 96 股）

### 结算

- 15min 周期结束，赢家侧每股 $1，输家侧 $0
- PnL = 赢家侧净持仓 × $1 − 净投入（总买入花费 − 总卖出回收）

## 七、盈亏归因（用于回测验证方向）

策略跑完后，每场应可归类到以下模式，用于诊断：

| 结果 | 判据 | 健康度 |
| --- | --- | --- |
| 盈利 | sum_avg < 1 且两侧净持仓对称 | ✅ 理想 |
| 亏损 A | sum_avg ≥ 1.05 | 买贵了，循环没压下来 |
| 亏损 B | 1.0 ≤ sum_avg < 1.05 | 边际亏损，波动不足 |
| 亏损 C | sum_avg < 1 但持仓不对称 | 收手对称补仓失效 |
| 低波动亏 | 场波动 < 0.4 | 无止盈机会，天然短板 |

对标 0xe00740bc：胜率 ~53%，赢场均 +52、亏场均 -35，场均 +11。我们的 BTC 数据波动更充足（98% ≥0.4），理论上有望达到或超过此表现，但趋势性更强是风险点。

## 八、与旧策略的区别

旧策略（已废弃）：只买不卖、被动扛结算、独立"动态对冲阶段"，933 场场均 -4.5、胜率 22%，亏损 100% 来自对冲。

新策略：高频买卖循环、逐笔 Maker 止盈、循环自然对称，靠波动赚差价。核心差异是**加了卖出能力和循环回收**——这是旧策略缺失的关键能力。
