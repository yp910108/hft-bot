# Polymarket BTC 15 分钟二元预测市场交易策略

## 一、核心思想

**赚钱靠对冲价差，不靠方向预测。**

在便宜的一侧梯度接低建仓，靠对面配对买单把双边均价之和（sum_avg）压到 1 美元以下，使两侧的结算盈亏同时转正，锁定绝对利润。头寸扛到 15 分钟交割结算自然兑现。**做市阶段盈利出口是「两侧结算盈亏都达标 → 收手扛结算」**；进入对冲后另有各自的收手出口（动态对冲的微利逃生、EV 的期望转正）。当单边裸露被行情拖入亏损时，启动对冲模块改善持仓结构。

二元期权结算刚性：胜方每股价值 1 美元，败方 0。

---

## 二、资金池

总资产 $V$ 在开盘时划分为四个独立资金池：

| 资金池 | 比例 | 作用 |
| --- | --- | --- |
| **备用金池** | 25% | 绝对禁止日常动用，仅极端防御下提供最后流动性。 |
| **核心做市池** | 15% | 做市阶段双边梯度铺单的唯一资金来源。 |
| **动态对冲池** | 22.5% | 动态对冲阶段（Maker 织网防守）的资金来源。 |
| **EV 对冲池** | 37.5% | EV 对冲阶段（Taker 扫盘）的资金来源。 |

资金重心在对冲：做市只用 15% 开小双边底仓，60% 留给对冲。

**全局现金哨兵**：任何模块下单前经资产审计器校验，可用现金不足时拒绝新开仓。

## 三、关键口径（全策略统一）

| 名称 | 公式 | 说明 |
| --- | --- | --- |
| **某侧结算 pnl** | $Q_{side} - C_{total}$ | 该侧若赢能拿回多少。$Q$=该侧净入仓股数（胜方每股值 1），$C_{total}$=双边累计净投入现金（含已发生手续费）。 |
| **某侧浮亏 pnl** | $Q_{side} \times bid_{side} - cost_{side}$ | 逐侧算，只对有持仓的侧算。用**本侧自己的成本**、用 **best_bid**（现在立即割肉能拿回多少）。 |
| **未配对保护成本** | $\max(0,\ Q_{up} - Q_{down}) \times \overline{P}_{up}$ | 主战场侧比对面多出来、没配对保护的裸露头寸现价值（用主战场侧成交均价）。 |
| **sum_avg** | $\overline{P}_{up} + \overline{P}_{down}$ | 双边成交均价之和。压到 1 以下则两侧结算 pnl 同时为正。 |

**为什么浮亏用 bid 不用 mid**：浮亏对冲是极端情况的底线防守，必须按「现在立刻强制市价割肉能拿回多少」算最坏情况，用盘口能立即吃单的 best_bid，绝不能用中间价掩盖流动性匮乏时的真实亏损。

## 四、全局参数

跨阶段共用的参数。各阶段专属参数见各自模块的参数表。

| 参数 | 值 | 说明 |
| --- | --- | --- |
| **主战场阈值** | best_ask < 0.5 | 只在便宜侧建仓 |
| **利润 margin** | 0.02 | 配对价留出的利润空间 |
| **续挂步长** | 0.01 | 主战场最低活跃挂单价下方续挂一档的偏移 |
| **亏损触发线** | −2%V | 全局共用的亏损线：做市阶段用它判穿线进对冲，动态对冲用它判继续对冲 |
| **单边持仓门槛** | 3%V | 结算 pnl 触发的对侧成本门槛 |
| **最大敞口上限** | 11.25%V（动态对冲池的 50%） | 单边裸露敞口的硬红线 |

**最大敞口为何挂钩对冲池的 50%**：建立「风险敞口」与「救火弹药」的硬链接。最极端单边行情下，系统永远保留 2 倍于当前裸露敞口的对冲资金可用，这是安全垫机制。

**+0.5% / −2% 不对称是有意为之**（+0.5% 见做市模块）：做市利润来自微小价差积累，0.5% 是合理离场点；触发对冲意味着进入防御态，Taker 4% 费率 + 滑点使对冲成本极高，防守线太紧（如 −0.5%）会在正常震荡中被反复洗盘、被手续费抽干，必须留 −2% 容忍市场噪音。

## 五、有限状态机

事件驱动。每个 tick 先过一遍**全局条件路由**（按优先级链从上到下裁决，见第十节），再进入当前阶段的细节逻辑。

```
全局优先级链（每 tick 先判）：
  时间红线(< 1min) > 熔断求生(Spread > 30%) > EV 对冲 > 动态对冲 > 核心做市

[场开始] ➔ 建仓态：铺初始三档梯度单
                 │
                 ▼ （首笔成交，哪怕 1 股）
            配对态（常驻）
                 │
                 ├─➔ [两侧结算 pnl 都 ≥ +0.5%V] ──────────────➔ 收手扛结算
                 ├─➔ [结算/浮亏 pnl 穿 −2%V] ─────────────────➔ 动态对冲
                 ├─➔ [TTE < 5min 且尾盘亏损破线] ──────────────➔ EV 对冲
                 ├─➔ [TTE < 5min 且尾盘未破线] ────────────────➔ 收手扛结算
                 └─➔ [核心做市池耗尽] ────────────────────────➔ 挂机等待
                          │
                          ▼  动态对冲（Maker 织网，补少仓侧摽齐）
                          ├─➔ [两侧结算 pnl 都 ≥ +0.25%V] ──────➔ 微利逃生·扛结算
                          ├─➔ [pnl ∈ (−2%V, +0.25%V)] ──────────➔ 观察态（静默等行情）
                          ├─➔ [Target 侧敞口撞红线] ────────────➔ 原地挂机（不发单·不升级 EV）
                          ├─➔ [资金耗尽：缺口仍大但本步预算买不起最低单] ─➔ CancelAll + 观察态（置资金耗尽标志·黏住不再重启对冲）
                          └─➔ [TTE < 5min 且 (双边负 2 次 或 尾盘破线)] ─➔ EV 对冲
                                   │
                                   ▼  EV 对冲（Taker 扫盘，顺势单边押注·战略翻转）
                                   ├─➔ [窗口 5 ~ 1min 且 概率 ∈ [0.75,0.85]] ─➔ 开火
                                   ├─➔ [EV > 0 / 概率 < 0.55 / 资金耗尽] ─────➔ 收手扛结算
                                   └─➔ [其他] ───────────────────────────────➔ 装死（不发单等行情进甜区）

任意阶段：
  Spread > 30% ──➔ 熔断求生态（CancelAll·稳定 5s 后重走全局路由）
  剩余 < 1min ──➔ 时间红线（无条件 CancelAll·锁死 Settlement_Wait）
```

## 六、做市模块

赚价差的主战场。在便宜侧梯度接低、对面配对摽齐，把 sum_avg 压到 1 以下，扛到结算双赢。

### 6.1 建仓态

1. 选 best_ask < 0.5 的一侧为主战场，两侧皆满足取更低者（例：开局 UP 建仓）。**一轮生命周期内绝对不换边**——状态机需要锚点，频繁切换会变成双边裸奔。
2. 以该侧 best_ask 为基准，向下平铺三层非饱和阵型：
   - 1 层：best_ask − 0.01，动用核心做市池 2%
   - 2 层：best_ask − 0.02，动用核心做市池 3%
   - 3 层：best_ask − 0.03，动用核心做市池 5%
3. 价格向下取整（不抬买价），股数向下量化后校验最小量（5 股且 ≥ $1），不够的档跳过（不上调，守资金纪律）。
4. 首笔成交（哪怕 1 股）立刻单向跃迁至配对态。

初始三档满打满算占 1.5%V（核心做市池 15% × 10%）。

### 6.2 配对态

常驻状态，直到利润锁定、对冲触发或最后 5 分钟。

**主战场侧（UP）成交 → 触发重算**

1. **检查最大敞口**：
   - 未超 11.25%V → 在主战场最低活跃挂单价下方一档（−0.01）补一档 UP 限价单，占核心做市池 2%。若主战场无活跃挂单（全成交），退化取均价为基准。目标价位已有我方挂单时不重复发单。
   - 已超 11.25%V → 撤销所有 UP 未成交挂单，**做市阶段永久停铺 UP**（哪怕之后 DN 配对成交把敞口降回去也不恢复，防趋势盘反复抖动）。备注：此停铺仅约束做市阶段，若熔断恢复后重新回到做市阶段且届时未超最大敞口，仍会继续铺 UP。
   - 敞口判定：未配对保护成本 + UP 活跃挂单总金额 + 拟发新单金额 ≤ 11.25%V。
2. **DN 侧铺配对单**：
   - 价格 = $\min(1 - \overline{P}_{up} - 0.02,\ ask_{down} - 0.01)$
   - 数量 = 主战场总持仓 − 对面已有总持仓（目标摽齐，差额 ≤ 0 不下单，根除滚雪球）

取 min 让配对价永远 ≤ 对面 ask − 0.01，既能立即排队又不牺牲利润，消除了「配对价高于 ask 只能挂起等待」的情况。

**配对侧（DN）成交 → 不触发重算**

配对侧成交意味着风险闭环和利润兑现。**绝不触发重新铺设主战场逻辑**（防无限套娃建仓），仅做两件事：
1. 更新本地账本和均价。
2. 触发检查全局「利润锁定」或「对冲触发」阈值。

**精细订单管理（UP 成交后 DN 配对单的处理）**

UP 成交导致 UP 均价变化，需重算 DN 最新配对价，对比当前在盘口排队的旧配对单：

- **配对价变化 ≤ 0.01**：不撤单。微小价差不足以覆盖重新排队的成交率损失，让旧单继续排队保住 time priority。
- **配对价变化 > 0.01**：撤旧单、重挂新单。均价已被大幅摊低，旧单挂得太低（利润空间留太大、难成交）或太高（成交会导致利润不达标），必须修正。
- **缺口变大但配对价 ≤ 0.01**（如原差 100 股、现差 150 股）：不撤原 100 股单，在相同价格追加发送 50 股新单。原排队优势完美保留，只在队尾多加 50 股。

### 6.3 退出条件

| 触发 | 动作 |
| --- | --- |
| **利润锁定** | 两侧结算 pnl 都 ≥ +0.5%V → 撤所有单收手扛结算 |
| **结算 pnl 穿线** | 任一侧结算 pnl ≤ −2%V **且** 引发亏损的对侧持仓成本 > 3%V → 动态对冲 |
| **浮亏 pnl 穿线** | 任一侧浮亏 pnl ≤ −2%V → 动态对冲 |
| **核心做市池耗尽** | 拟挂金额 + 已挂金额 + 已成交金额 ≥ 15%V，或剩余不够最小单 → 挂机等待（资金耗尽被动停手，留在做市态等已挂单成交，不进对冲） |
| **最后 5 分钟（尾盘规则）** | 见下方「全局尾盘规则」 |

### 6.4 做市阶段参数

| 参数 | 值 |
| --- | --- |
| 利润锁定线 | +0.5%V |
| 亏损触发线 | −2%V（引用全局） |

## 七、动态对冲模块

被动急救态。Maker 织网把双边摽齐、把 sum_avg 拉回 1 以下。受敞口红线约束，撞线即原地挂机，绝不无限加仓。

### 动态对冲的目的（明确定义）

- **目的**：补齐**空仓 / 少仓侧（Target Side）**，把双边摽齐，让 sum_avg 有机会压回 1 以下。sum_avg 偏高的根因通常就是双边没摽齐（一侧重仓、对面太少）。
- 每向 Target Side 发一笔单，Target Side 的敞口就增加一点。
- **敞口红线的作用**：当 Target Side 的「已有持仓 + 活跃挂单 + 拟发新单 > 最大敞口上限」时，说明这一侧也买够了（防止补仓补成新的单边重仓）。此时**停止发单（Skip），在动态对冲态原地挂机**——不撤单、不跳转。
- **绝不能因为「撞了敞口红线」就去触发 EV 对冲**。继续升级到更激进的 Taker（EV）等于继续加仓送死。EV 有它自己独立且严苛的触发条件（见第八节）。

### 进入

配对态满足以下任一条件触发（与做市退出条件一致）：
- 任一侧结算 pnl ≤ −2%V **且** 引发亏损的对侧持仓成本 > 3%V
- 任一侧浮亏 pnl ≤ −2%V

进入时：
- 撤销做市阶段所有挂单。
- 开启「双边都负」独立计数器。

### 补哪边（Target Side）

补**持仓少的那侧**（空仓 / 少仓侧）去摽齐。哪侧股数少就补哪侧。**当双边股数基本相等时，说明敞口已平齐、对冲任务阶段性完成，本步执行 Skip（跳过发单）**——此时若利润达标（两侧结算 pnl 都 ≥ 微利逃生线 +0.25%V）则撤单扛到结算，否则原地挂机等待，直到盘面成交导致股数再次失衡（重新补少的那侧）或满足条件升级 EV 对冲。

### 单步织网（带摽齐 Cap）

没有「轮」的概念，一步一步持续进行。每次准备织网前，先算双边股数缺口，按下面的顺序判定：

**第一步：缺口是否已平齐（粉尘缺口）**

若双边股数缺口 ≤ 0，或缺口太小、按它下单量 / 金额低于交易所最低要求（5 股且 ≥ $1）→ 判定为**粉尘缺口**，逻辑上**等同于 Delta Neutral 已达成**（够不到的零头不纠缠）。此时本步 Skip，按「补哪边」里平齐后的处置走（利润达标 +0.25%V 则撤单扛结算，否则挂机等失衡 / 升级）。

**第二步：是否资金耗尽**

缺口仍大，但**本步预算（对冲池剩余 × 7.5%）连交易所最低单都买不起** → 判定为**资金耗尽**：没钱补了，对冲已是空谈。立即 **CancelAll，进观察态，并置「资金耗尽」标志位**——该标志黏住本场对冲（对冲池只减不增），观察态里不再因 pnl ≤ −2%V 反复重启对冲，纯静默等行情自然修复或满足条件升级 EV。

**第三步：正常织网（带 Min 摽齐 Cap）**

单步下单量用 Min() 安全阀约束：

$$单步下单量 = \min(\ 本步资金预算可买股数,\quad 双边股数缺口\ )$$

- **本步资金预算** = 对冲池**当前剩余** × 7.5%，决定这一步最多买得起多少。
- **双边股数缺口** = 双方股数之差，决定最多缺多少（绝对上限）。

终极目标是 Delta Neutral（双边股数绝对相等）。一旦该笔订单成交后能让双边股数平齐，多余的资金预算必须立刻截断退回资金池——绝不允许「按资金计划盲目执行」导致过度对冲（Over-hedging），在反方向制造出新的裸露敞口。

**按 Min() 取谁，决定怎么挂（降维聚拢）：**

- **Min 取「资金预算」**（钱不够、缺口大）：严格按 40%/30%/30% 铺三层 Maker 梯度，精打细算分步蚕食。
  - 1 层：best_ask − 0.01，占本步资金 40%
  - 2 层：best_ask − 0.02，占本步资金 30%
  - 3 层：best_ask − 0.03，占本步资金 30%
- **Min 取「股数缺口」**（钱多、触发摽齐 Cap，对冲收尾阶段）：放弃三档拆分，将截断后的总缺口股数 **100% 全部挂在第一档（best_ask − 0.01）**一把摽平，多余预算退回池。

**差额追加机制（同价位不重复铺、但补足缺口）：**

同价位已有活跃挂单时**不整档跳过**，而是计算差额追加：
- 本档应挂股数（三档模式 = 该档预算/价格，摽齐 Cap 模式 = 缺口总量）。
- 该价位活跃股数 = 该侧该价位所有活跃单的 qty 之和。
- 增量 = 本档应挂股数 − 该价位活跃股数。
- 增量 > 最小下单量 → 在该价位**追加一笔增量单**（旧单保留不动、排队优势不丢）。
- 增量 ≤ 最小下单量 → 不发（粉尘差额不纠缠）。

这和配对态精细订单管理的「缺口变大追加」是同一思路：保住旧单排队位置，只在队尾补足增量。避免因粉尘旧单占坑导致大额缺口迟迟发不出新单。

### 单步生命周期（三种结束）

- **全成交**：本步 Maker 全部成交（理想），记时间戳 s1。
- **超时（3 秒）**：到达最长生命周期，不管成没成交强行结束。由**本地 wall-clock 定时器**兜底触发——发单瞬间启动 3 秒异步定时任务，全成交则取消；3 秒到仍未全成交则强制介入：按撤单规则撤未成交单、宣告本步超时结束、记 s1。保证缺 tick 时不死锁。
- **价格偏离**：挂了 best_ask − 0.01 的 Maker，但市场暴跌，best_ask 瞬间偏离挂单价超 0.03，Maker 已成深海死单毫无防守意义，立即判本步失效结束。

### 冷却（1 秒）

两步之间 1 秒冷却，用**时间戳卡点被动校验**（不起定时器主动唤醒）。本步结束记 s1，此后的 tick 若 `Now − s1 < 1s` 直接 return；满 1s 后的某个 tick 才真正发起下一步。实际 >1s（要等下一个 tick 到来）。

### 每步开始前

检查 Target Side 未成交 Maker 单偏离 best_ask 是否 > 0.03，是则精确撤销。深海死单会占用对冲预算池额度，不撤就算不出下一步的钱。

### 状态流转（以结算 pnl + 缺口为基准）

进入动态对冲的触发线（−2%V）和退出的修复线（−1%V）**不同**——形成迟滞带，避免"刚脱线就停、带着敞口亏损扛到结算"。

| 条件 | 动作 |
| --- | --- |
| pnl ≥ +0.25%V | **微利逃生** → 撤所有单转结算态扛交割（不管缺口，利润到手就跑） |
| pnl ≥ −1%V **且** 双边股数缺口 ≤ 最小下单量 | **修复完成，进入静默挂机**：停发新单、不撤已挂单、等利润或等升级 |
| 其他（pnl < −1%V，或 pnl ≥ −1%V 但缺口仍大） | **继续织网**（在 Target Side 补缺口，每步 1 秒冷却） |

修复完成的判定必须**同时满足两条**（AND）：
- ① **pnl ≥ −1%V**（亏损已修复到可接受范围）
- ② **双边股数缺口 ≤ 最小下单量**（敞口已平齐，视同 Delta Neutral）

只满足一条不停手：pnl 好了但缺口大 → 继续补；缺口补齐了但 pnl 还差 → 继续等成交拉回 pnl。

静默挂机不是「免疫升级」：双边负计数、尾盘规则仍在跑，满足照样跳 EV 或扛结算。

### 升级 EV

满足以下任一即进 EV：
1. **双边都负 2 次**（边沿触发）→ **不限 TTE，立即进 EV**。双边负 2 次是极其强烈且罕见的信号——sum_avg > 1 且两侧都被高位套牢，Maker 织网已证明失败。早进 EV 能在优势方价格还没飙到天上时出手，覆盖沉没成本的机会更大。计数器作为跨阶段全局量，由 engine 统一维护，熔断等状态跳转不会抹掉。
2. **尾盘亏损破线**（TTE < 5min 且亏损大侧结算 pnl ≤ −2%V）：见下方「全局尾盘规则」。

注意：**敞口撞红线不是升级 EV 的路**——撞红线只在 Target Side 原地挂机（见上「动态对冲的目的」）。

### 其他退出

- **微利逃生**：两侧结算 pnl 都 ≥ +0.25%V → 撤所有单收手扛结算。
- **资金耗尽**：缺口仍大但本步预算买不起最低单 → CancelAll + 进观察态 + 置「资金耗尽」标志位（黏住本场，不再重启对冲；详见「单步织网」第二步）。

### 动态对冲阶段参数

| 参数 | 值 |
| --- | --- |
| 微利逃生线 | +0.25%V |
| 修复线 | −1%V（pnl 修复到此线以上 + 缺口平齐才允许静默） |
| 亏损触发线 | −2%V（引用全局） |
| 单步资金 | 对冲池剩余 × 7.5% |
| 单步生命周期 | 3 秒（wall-clock 兜底） |
| 单步冷却 | 1 秒 |
| 深海死单偏离阈值 | 0.03 |

## 八、EV 对冲模块

终极止血态。当常规对冲破产、比赛进入尾声（< 5min）且趋势已确立时，系统**认输止血**。此时系统实现**战略意图的彻底翻转**：不再追求双边敞口的绝对平齐（Delta Neutral），而是转为**「动量顺势单边押注」**——动用最大的一笔钱（EV 对冲池 37.5%V），以 Taker 模式强行追买**真正会赢的优势方**（落在 0.75 ~ 0.85 甜区侧），试图靠交割时拿到的刚性胜方收益（每股 1）来覆盖全局的巨额沉没成本。**从「追求平齐的被动防守」跃迁为「追求期望值的单边搏命」，才是 EV 对冲的核心灵魂。**

### 数学期望公式

$$EV = p \times up\_settle\_pnl + (1-p) \times down\_settle\_pnl$$

- $p$ = UP 侧 Mark Price（盘口中间价，近似 UP 胜出概率）。
- up/down_settle_pnl 用**结算 pnl 口径**（$Q_{side} - C_{total}$，含双边总成本），绝不用浮亏口径——EV 算的是「现在花钱吃货，到 15 分钟交割时系统整体的期望收益」。

### 进入

满足以下任一信号即进 EV：
1. **双边都负 2 次**（边沿触发，动态对冲专属）→ **不限 TTE，立即进 EV**。
2. **尾盘亏损破线**（TTE < 5min 且亏损大侧结算 pnl ≤ −2%V）。

两个信号都不满足且 TTE < 5min → 收手撤单扛结算。

进入时立刻 **Cancel All** 撤销所有挂单，盘面清零（之后是「空仓挂单、只有已成交持仓」状态）。

### 战略进入 ≠ 战术开火（关键解耦）

「进入 EV 状态」和「打出 EV 子弹」是两码事。进来后先评估优势方概率，只有落在甜区才真正出手，否则装死（不发任何 API 请求、留在 EV 态等行情进甜区）。

### 出手判定（分时段甜区）

| 剩余时间（TTE） | 优势方概率出手甜区 |
| --- | --- |
| > 5min | [0.60, 0.75] |
| 5min ~ 1min | [0.75, 0.85] |
| < 1min | 时间红线接管（强制收手） |

越临近交割流动性越差、要求的确定性越高（甜区上移）。早期（>5min）优势方还在 [0.60,0.75] 时价格更便宜，用同样的钱能买更多股，覆盖沉没成本的机会更大。

| 优势方概率 | 动作 |
| --- | --- |
| ∈ 当前时段甜区 | **开火**：IOC Taker 追买优势方（前提：盘口 best_ask 存在且 ≤ 保护价 0.85，否则装死等盘口有合适的货，不触发冷却） |
| ∈ [0.55, 甜区下界) | **装死**：不够确定，不发单等行情 |
| > 甜区上界 | **装死**：价格极贵（滑点大）+ 共识极度趋同，用 4% Taker 追买盈亏比恶劣，不做冤大头；若一去不回头硬挺到结算就扛到结算收割 |
| < 0.55 | **反转退出**：判定期望反转，立即收手扛结算 |

「装死」只是引擎这一刻不发 API 请求；FSM 没死——每个 BookUpdate 都重算 EV、概率区间、反转线，满足出手或退出条件就在下一个 tick 瞬间响应。

### 单步

- 没有「轮」的概念，一步一步持续进行。
- 单步金额 = **EV 对冲池当前剩余**的 25%。
- **强制 IOC 模式**：发价格上限为 0.85 的限价单，撮合引擎吃掉 best_ask 到 0.85 之间所有流动性，未成交部分立即撤销。绝不允许挂成 Maker 被动等成交。
- 单步生命周期：IOC 在同一原子事务里返回 Fill（成交）+ Cancel（未成交）回报，收到回报本步即自然闭环，不需要 wall-clock 定时器兜底。
- 两步之间 **2 秒冷却**（时间戳卡点被动校验）。

### 退出（任一满足即撤所有单扛结算）

- **EV > 0**：期望转正，目的达成。
- **概率跌破 0.55**：判定期望值反转，立即收手。
- **资金耗尽**：EV 池剩余不够最小下单量或金额（按保护价 IOC Limit Price 0.85 计算）。
- **剩余 < 1min**：时间红线强制收手（见第十节）。

### EV 对冲阶段参数

| 参数 | 值 |
| --- | --- |
| 进入条件 | 双边负 2 次（不限 TTE）/ 尾盘破线（TTE < 5min） |
| 出手甜区（TTE > 5min） | 优势方概率 [0.60, 0.75] |
| 出手甜区（TTE 5min ~ 1min） | 优势方概率 [0.75, 0.85] |
| 反转退出线 | 0.55 |
| 单步金额 | EV 池剩余 × 25% |
| IOC 保护上限价 | 0.85 |
| 单步冷却 | 2 秒 |

## 九、熔断求生态（Circuit_Breaker_Survival）

流动性崩溃时的全局保护。不属于任何交易阶段，是凌驾于做市/对冲之上的安全中断。

### 触发指标：买卖价差率

$$Spread\_Ratio = \frac{best\_ask - best\_bid}{best\_bid}$$

正常 15 分钟盘 Spread_Ratio 维持极低（约 0.01~0.03）。**任一侧** Spread_Ratio > 30% 即触发全局熔断（例：bid 掉到 0.20、ask 挂 0.55）——说明理性做市商已撤单跑路，流动性崩溃。

**为何任一侧都看**：UP/DN 价格强镜像耦合（UP + DN ≈ 1）。非主战场侧突然 spread 爆宽不是孤立事件，意味着整个定价机制崩溃（预言机故障 / 巨鲸撤资 / 内幕泄露）。流动性崩溃是全局信号，任一侧异常都必须立刻全局熔断，否则会被单边毒流穿透。

### 执行动作

立刻 Cancel All，FSM 切换至熔断求生态。

### 防抖恢复（迟滞）

必须 Spread_Ratio < 10% **且持续稳定 5 秒以上**才解除熔断。不加这 5 秒迟滞，机器会在 spread 于 30% 上下横跳时疯狂发单又撤单，浪费 API 速率额度甚至触发风控封禁。

### 恢复即重新评估（带记忆的全局重评）

熔断恢复**不是回到断点继续**（闪崩期间价格、概率、TTE 已剧变，刻舟求剑会出错），而是：
1. **保留状态变量**：双边亏损计数、总持仓成本、各资金池剩余预算等底层计数器和标记全部保留。
2. **挂单已清零**：熔断时已 Cancel All，恢复时是「空仓挂单、只有已成交持仓」。
3. **重走全局条件路由**（见第十节）：把最新盘面切片丢进路由从上到下重判，决定进结算 / EV / 动态对冲 / 退回做市。主战场不重选，沿用已锁定的那一侧。

## 十、全局规则

### 绝对优先级链（Absolute Priority Chain）

任何 tick 都先过一遍优先级链，从上到下裁决，高优先级无条件压制低优先级：

```
时间红线 (<1min) > 熔断求生 (Spread>30%) > EV 对冲 > 动态对冲 > 核心做市
```

这是**全局条件路由（Global Evaluation Router）**的实现：所有「同时满足多个条件」的歧义都靠这条链裁决。

### 全局尾盘规则（做市态 + 动态对冲态共用）

剩余时间 < 5min 时，所有交易阶段统一按此处置：
- 若**亏损大侧的结算 pnl ≤ −2%V** → 强制进 EV 对冲。
- 否则 → 收手撤单扛结算。

不使用概率加权——痛就是痛，只要最坏情况亏损超线就触发，和做市阶段进动态对冲的触发口径一致。

最后 5 分钟波动剧烈、流动性极差，Maker 单大概率排不到队或瞬间成深海死单，动态对冲在此彻底失效，只能 Taker 扫盘或躺平扛结算。

### 时间红线（< 1min 硬中断）

剩余时间 < 1min 时拥有系统最高优先级。无论当前处于做市、动态对冲、EV 对冲还是熔断求生的哪个阶段，**无条件切断当前执行流，立刻 Cancel All，状态锁死在 Settlement_Wait（等待结算态）**，再不恢复。

生存哲学：此时任何挣扎的期望值都是负的，带现有持仓硬扛到结算（哪怕亏损）也绝对优于在最后几十秒被超大滑点双杀。

时间红线 > 熔断恢复：若熔断 5 秒防抖期未数完就到了 < 1min，时间红线强行接管、丢弃 5 秒计时器、直接锁进 Settlement_Wait。

### Maker 挂单价规则（做市态 + 动态对冲态）

**Maker 单挂单价**规则：
- 挂单价 < best_ask → 直接挂。
- 否则 → 以 best_ask − 0.01 挂。

目的：防止做市时意外成为 Taker 被收 4% 手续费。**仅适用做市态和动态对冲态的 Maker 单**，EV 阶段的 Taker 单不受此约束（EV 本就主动做 Taker，直接 IOC 设保护上限价 0.85）。

---

## 十一、关键红线

- **绝不主动止损**：除熔断求生与时间红线外，做市阶段持仓严禁割肉，靠对冲调整结构或扛到结算。
- **绝不触碰备用金池**：EV 对冲也只动用 EV 对冲池，25% 备用金池绝不动用。
- **高频账本高可用**：每笔成交原子级落盘（WAL），panic 重启可从本地 JSONL 完美复原。均价为精确值不做舍入。

## 十二、手续费模型

- Taker 费率 ~4%（按 4% 计提，略偏保守），Maker 费率 0%。
- 扣费口径：到手股数扣减（下单 100 股价 0.5 费率 4% → 实付现金 50、净入仓 96 股）。
- 费率可配置，由 `FeeModel` 统一管理。
