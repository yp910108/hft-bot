## 开发阶段

### 阶段 1：domain 地基

- TimeInForce 枚举（Gtc/Ioc），Order 加该字段。
- PnL 口径重做：`settle_pnl(side)` = Q_side − C_total；`float_pnl(side)` = Q_side × bid_side − cost_side（逐侧、用 bid）。
- 移除旧的 breached_side 等与旧触发耦合的方法，按新口径重建。
- RobotState 重定义为 7 状态（见阶段 3）。
- 保留并复用：types、fee、market（mark_price 中间价）、OrderConstraints、Generation/OrderId。
- 单测：每个口径公式给定持仓与盘口断言数值。

### 阶段 2：ledger 按侧可查

- 账本提供按侧查成交均价、成交成本、持仓量的接口（小策略口径计算要用）。
- WAL 落盘与重放复原保持。
- 单测：建仓/配对成交后按侧查值正确；重放复原一致。

### 阶段 3：fsm 状态与合法转移

- 7 状态：建仓态 Building、配对态 Pairing、动态对冲 DynamicHedge、观察态 Observing、EV 对冲 EvHedge、熔断求生 CircuitBreaker、结算等待 SettlementWait。
- 仅定义状态与「合法转移校验」（哪些跳转允许），**不含决策逻辑**（决策在 strategy::router）。
- 计数器（双边负次数等）作为状态携带数据或独立结构。
- 单测：合法转移通过、非法转移被拒。

### 阶段 4：risk 四池 + 敞口

- CapitalPools 改四池（reserve 25% / grid_maker 15% / dynamic 22.5% / ev 37.5%），构造校验和为 1。
- Cash Guard 保持。
- 新增最大敞口校验：未配对成本 + 活跃挂单金额 + 拟发新单 ≤ 11.25%V。
- 单测：四池分配、Cash Guard 拦截、敞口超限拦截。

### 阶段 5：exchange — Clock + Simulator + 生命周期

- Clock trait（now），虚拟时钟实现（时间戳推进）。
- backend trait 扩展：撤单经事件异步回报；新增「撤单失败」事件变体。
- Simulator：Maker 保守填单；Taker IOC 吃 best_ask 到保护价 0.85、未成交即撤；接入 Clock。
- 订单生命周期：Active / CancelPending / Filled / Canceled。
- 单测：Maker 穿越成交、IOC 部分成交+撤销、撤单/成交竞态、虚拟时钟推进。

### 阶段 6：strategy 共享类型与 router

- context.rs：DecisionContext（只读世界切片）、Decision（指令 + 跳转请求）。
- router.rs：优先级链路由（时间红线 > 熔断 > EV > 动态 > 做市），依据 DecisionContext + FSM 状态选中小策略。
- PhaseStrategy trait。
- 单测：每条优先级分支在构造的 context 下命中正确的小策略。

### 阶段 7：strategy 建仓态 + 配对态

- building.rs：选主战场（ask<0.5 更低者，锁定不换边）、铺三档、首笔成交跳配对。
- pairing.rs：主战场成交触发重算（敞口检查→续挂/永久停铺）、配对单价/量、精细订单管理（偏差≤0.01 不撤 / 缺口变大追加 / 偏差>0.01 撤重挂）；配对侧成交不触发重算但更新+查阈值；Maker 挂单价规则。
- 退出：利润锁定 +0.5%V、结算/浮亏穿线、池耗尽挂机、最后 5 分钟。
- 单测：布阵、续挂、配对价 min 公式、摽齐量、三种精细订单分支、永久停铺、各退出触发。

### 阶段 8：strategy 动态对冲态 + 观察态

- dynamic_hedge.rs：补**少仓侧摽齐（Delta Neutral）**、单步带 **Min() 摽齐 Cap + 降维聚拢**（Min 取预算→三档 40/30/30 / Min 取缺口→全挂第一档）、粉尘缺口视同已平齐、平齐后查利润 +0.25%V（达标扛结算/否则挂机）、**资金耗尽** → CancelAll + 观察态 + 置标志位黏住、敞口撞红线 → **原地挂机 Skip（不升级 EV）**、深海死单撤销、双边负计数、单步生命周期(超时/全成交/偏离)、1 秒冷却。
- 观察态：区间 (−2%, +0.25%) 静默等行情，升级检查仍跑；资金耗尽标志置位后不再因 pnl≤−2%V 重启对冲。
- 单测：补哪边（少仓侧）、Min Cap 两种分档、粉尘缺口视同平齐、平齐→查利润/挂机、资金耗尽黏住、敞口红线挂机不弹 EV、双边负 2 次（须 TTE<5min 才进 EV）、观察态边界、冷却卡点、深海死单撤销。

### 阶段 9：strategy EV 对冲态 + 熔断求生态

- ev_hedge.rs：EV 公式（结算口径）、**进入大门 TTE<5min（硬前提）**、出手甜区统一 **[0.75, 0.85]**（TTE 5~1min）、装死（概率不在甜区不发单等行情）、IOC@0.85 单步 25%、2 秒冷却、退出（EV>0 / 跌破 0.55 / 资金耗尽 / <1min）、战略翻转：从 Delta Neutral 被动防守 → 动量顺势单边押注。
- circuit_breaker.rs：Spread_Ratio>30% 任一侧触发、CancelAll、<10% 稳定 5 秒恢复、恢复即重走 router（带记忆）。
- 时间红线：<1min 最高优先级硬中断 → CancelAll 锁 SettlementWait。
- 单测：EV 进入大门 TTE<5min、统一甜区 [0.75,0.85]、装死分支、反转退出、熔断触发/恢复迟滞、时间红线压制一切。

### 阶段 10：engine 单写者外壳

- event_loop.rs：取事件→更新事实→组 context→router→应用 Decision→落盘。
- book.rs：活跃挂单簿 + 订单生命周期，以回调为准。
- timer.rs：定时器到期注入 TimerFired。
- apply.rs：Decision 过风控→发指令→更新本地镜像。
- journal.rs：事件溯源落盘（输入事件 + 输出指令）。
- 状态追踪：engine 维护 `funds_exhausted: bool` 资金耗尽标志位，置位后观察态不再因 pnl≤−2%V 重启对冲。
- 单测：单写者串行处理、定时器注入、挂单簿生命周期、风控拦截、资金耗尽标志黏住、事件重放一致。

### 阶段 11：backtest 串联验证

- 虚拟时钟接 real_data 时间戳，喂 Simulator + Engine 跑完整一场。
- BacktestReport 加 final_state 分布、各阶段触达率。
- 批量统计 933 场，输出 PnL / 胜率 / 阶段分布。
- 验证：跑通、可复现、产出 JSON。

### 阶段 12：app 装配

- 四池配置 + 全部阈值进 config/main_config.toml，启动校验 [fee] 段。
- 真实时钟 + 后端装配（实盘后端后置）。
