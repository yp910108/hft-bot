## 开发阶段

### 阶段 1：domain 地基

**目标**：领域类型就位，买卖双向语义与结算公式正确。

- `Phase` 枚举四阶段：Building / Cycling / Harvesting / Settled，线性推进校验（内联 `is_legal_transition`）。
- `Fill` 买卖现金流语义明确：买入 cash=付出、卖出 cash=回收。
- `OrderConstraints` 新增卖单价格向上量化 `quantize_price_up`（不降低卖价），保留买单向下量化。
- 结算：`settle_pnl(net_position, winner)` = 赢家净持仓 − 净持仓成本。
- 保留复用：types（Price/Qty/Money/Side/OrderRole/OrderDirection/TimeInForce）、fee（Maker 0/Taker 4%）、market（BookTop/MarketSnapshot/mark_price）、Order/OrderId/Generation。
- **单测**：Phase 转移合法性；买卖 Fill 现金流；卖单向上量化；结算公式给定持仓断言数值。

### 阶段 2：inventory 逐笔账本

**目标**：逐笔持仓模型就位，这是 LIFO 止盈的基石。

- `Lot { lot_id, buy_price, qty, opened_at }`、`LotId` 单调递增标识。
- `SideInventory { lots: Vec<Lot> }`，`Inventory { up, down, realized_pnl, cash_out, cash_in }`。
- 核心操作：`open_lot` / `close_lot` / `partial_close` / `net_qty` / `net_cost` / `net_avg` / `sum_avg` / `open_lots` / `net_invested` / `snapshot`。
- 已实现盈亏与现金进出累计。
- **单测**：开仓建 Lot；平仓算已实现盈亏；部分平仓减 qty；LIFO 顺序；净均价/sum_avg 计算；净投入 = cash_out − cash_in；卖超持仓截断。

### 阶段 3：exchange 买卖双向撮合

**目标**：Simulator 支持卖单，补齐旧架构缺口。

- `ExchangeBackend` trait（submit/cancel/cancel_side/cancel_all）+ `ExchangeEvent`（BookUpdate/Filled/Rejected/Canceled/CancelFailed）。
- Simulator 撮合规则（第一版穿越即成交）：
  - Maker 买单：best_ask < 挂单价 → 成交
  - Maker 卖单：best_bid > 挂单价 → 成交
  - Taker 买单：best_ask ≤ 限价上限 → 以 best_ask 成交
  - Taker 卖单：best_bid ≥ 限价下限 → 以 best_bid 成交
- Fill 现金流：买入 cash=名义、卖出 cash=回收（扣 Taker 费）。
- **单测**：四类订单各自成交/不成交边界；卖单 bid 穿越成交；Taker 卖单以 bid 成交；撤单/撤单失败回报。

### 阶段 4：strategy 三阶段纯函数

**目标**：三阶段决策逻辑，纯函数、可单测。

- `DecisionContext`（trigger/progress/market/inventory/active_orders/free_cash/constraints/config）、`Decision`（commands/transition）、`CommandIntent`（SubmitBuy/SubmitSell/Cancel/CancelSide/CancelAll）。
- `StrategyConfig`（entry_second/lot_qty/building_end/harvest_start/settle_start/tp_by_quartile/sl_by_quartile + 第二版可选项默认关闭）。
- 三阶段策略：
  - **BuildingStrategy**：双边挂 Maker 买单（bid，优先便宜侧，单笔 lot_qty）
  - **CyclingStrategy**：止盈扫描（open_lots 逐笔挂 Maker 卖单）+ 止损扫描（撤止盈、Taker 卖）+ 补买
  - **HarvestingStrategy**：停买、继续止盈、settle 后 Skip
- `tp(progress)` / `sl(progress)` 分段函数。
- **单测**：每阶段给定 context 断言产出的 commands；止盈触发条件（bid ≥ buy_price+tp）；止损触发（buy_price−bid ≥ sl）；分段 tp/sl 取值；建仓优先便宜侧；防重复挂止盈单。

### 阶段 5：engine 单写者编排

**目标**：把 domain/inventory/exchange/strategy 串成完整事件循环。

- Engine 持有状态（phase/inventory/book/id_gen/generation/market/sell_order_lot/lot_sell_order）。
- `handle_event`：apply_event_facts → decide → apply_decision。
- **apply_event_facts**：买入 Fill 开 Lot、卖出 Fill 查关联平 Lot、撤单清理关联。
- **decide**：算 progress、按 progress 路由、组装 context。
- **apply_decision**：SubmitBuy 过现金哨兵、SubmitSell 登记 Lot 关联、Cancel 标记、transition 更新 phase。
- 现金哨兵内联：`free_cash = 总资金 − 净投入 − 活跃买单名义`。
- OrderBook 生命周期（沿用旧设计：Active/CancelPending、部分成交、全成交移除）。
- **单测**：开场首 tick 双边挂单；买入成交开 Lot；涨价挂止盈卖单；卖单成交平 Lot 记已实现盈亏；跌价止损；progress 阶段切换；现金哨兵拦截；Lot↔卖单关联正确。

### 阶段 6：backtest 回测 + 结算

**目标**：933 场能跑通，PnL 正确。

- 沿用逐秒驱动骨架（on_market → 事件喂 engine → 指令下发 sim）。
- PnL = `inventory.realized_pnl + settle(净持仓, winner)`。
- MatchResult 加字段：已实现盈亏、结算盈亏、净投入、sum_avg、最终净持仓、买卖笔数、最深阶段。
- 沿用 CSV 加载（load_market/load_dir）。
- **单测**：单场跑通不 panic；PnL = 已实现 + 结算；震荡场有循环止盈；结算公式验证。

### 阶段 7：报告 + 批量回测

**目标**：933 场批量报告，验证策略表现。

- 批量回测 example：跑 933 场，汇总胜率/场均/赢亏场均/盈亏归因五类/波动分组。
- HTML 报告：逐场 PnL、逐笔操作序列（SELL→对面 BUY 展示层转换）、sum_avg 演变、波动分组表现。
- 展示层转换：`SELL side@price` → `BUY side.opposite()@(1−price)`，标注 converted，不影响底层数据。
- **验证**：对标 0xe00740bc（胜率 ~53%、场均 +11），我们 BTC 数据波动更足（98%≥0.4），看实际表现。

### 阶段 8：调参与诊断

**目标**：基于回测结果校准参数。

- 参数扫描：tp/sl 分段值、lot_qty、阶段边界。
- 诊断亏损场归因（五类占比），定位主要亏损来源。
- 评估是否启用第二版可选项（库存约束、对称补仓）。
- **不擅自调参**：每轮调参方向先与用户确认。
