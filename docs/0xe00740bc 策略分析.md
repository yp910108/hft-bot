## 策略核心逻辑

### 一句话

**开场即入，持续 15 分钟双边建仓 + 场内低买高卖，利润靠"卖出差价"压低 sum_avg，不靠方向预测。**

### 操作模型

```
┌─ 开场(前30秒) ────────────────────────────────────────────────┐
│ 97% 的场次在窗口开始 2 分钟内首笔操作                            │
│ 快速建立初始双边仓位(买 UP + 买 DN)                             │
└──────────────────────────────────────────────────────────────────┘
       ↓
┌─ 持续 ~12 分钟 ──────────────────────────────────────────────────┐
│ 每 2 秒一笔操作(中位间隔), 机器人全自动                           │
│                                                                   │
│ 核心循环:                                                         │
│   价格涨了 → 卖出涨了的一侧(54%止盈) → 等价获得对面便宜持仓       │
│   价格跌了 → 继续买入跌的一侧(抄底补仓)                           │
│   割肉卖出 → 31% 的卖出是亏损的(控制损失)                         │
│                                                                   │
│ 买入: 前期多(74%), 后期少                                         │
│ 卖出: 前期少(26%), 后期多(46%) — 越到后面越在"收割"              │
└──────────────────────────────────────────────────────────────────┘
       ↓
┌─ 交割 ───────────────────────────────────────────────────────────┐
│ 15 分钟到期, 按 BTC 最终价 vs 基准价结算                          │
│ sum_avg 中位 0.98 < 1 → 无论哪边赢大概率盈利                     │
└──────────────────────────────────────────────────────────────────┘
```

## 对我们策略的启示

1. **被动等 vs 主动做**：我们挂单等人砸，他主动出击利用波动。15 分钟内价格波动 0.58 是巨大的可利用空间，但被动 Maker 只能吃到穿越那一下（0.01），错过了中间所有波动。

2. **不卖出 = 放弃一半机会**：他 38% 的操作是卖出（止盈），这是他把 sum_avg 从 1.007 压到 0.98 的核心手段。我们的策略完全没有卖出能力。

3. **对称性靠主动维护**：他的对称度全局 0.91 是主动低买高卖的结果；我们靠被动配对实现对称，趋势场配对失败就裸露。

4. **高波动场才是利润来源**：他 97% 的利润来自高波动场（>0.7），而高波动场恰恰是我们亏得最惨的场——因为趋势场我们单边裸露。同样的高波动，对他是机会（更多低买高卖空间），对我们是灾难（配对失败）。
